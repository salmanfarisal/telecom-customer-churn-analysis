# 📉 Telecom Customer Churn Analysis
### Business Intelligence & Root Cause Analysis using Python

## 🌟 Executive Summary
This project analyzes a dataset of 7,043 customers to identify why users are leaving the service (Churn). I discovered that the company is losing **$139,130.85 per month** in revenue, primarily driven by Month-to-Month contracts and Fiber Optic users.

## 🛠️ Tech Stack
* **Python**: For data processing and analysis.
* **Libraries**: `Pandas` (ETL), `Seaborn` & `Matplotlib` (Data Visualization).

## 📂 Modular Pipeline Structure
I organized the analysis into 4 distinct scripts to ensure code maintainability:
1. **Data Cleaning**: Handled missing values in `TotalCharges` and performed type conversion.
2. **Driver Analysis**: Identified a **40% churn risk** among Fiber Optic and Month-to-Month users.
3. **Financial Impact**: Quantified the monthly revenue loss to provide business context.
4. **Correlation Study**: Generated a heatmap to visualize the relationship between Tenure, Charges, and Churn.

## 💡 Key Business Insights
* **Tenure is Key**: Longer-term customers (High Tenure) show significantly lower churn rates.
* **Contract Risk**: Month-to-month contracts are the #1 predictor of churn.
* **Revenue at Risk**: Targeted retention programs for Fiber Optic users could save up to $139k/month.
